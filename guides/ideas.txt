Overview
This document provides a high-level, code-free todo list for migrating from the current DREDGE system to the full connected domain reasoning architecture described in the evolution plan. Each item is a discrete milestone with estimated effort and dependencies.

Phase 1: Foundation — Zero-Code Domain Addition
Goal: Domains can be added without code changes
Duration: 2-3 weeks

1.1 Database Foundation
Create the hybrid query_results table migration

Migrate existing crime and weather data to new table

Create Domain and DataSource models

Add PostGIS extension for spatial queries (if not already)

Write migration for domain relationships table (future-proofing)

Add indexes on frequently queried fields

1.2 Registry Rewrite
Convert in-memory domain registry to database-backed loader

Implement hot-reload on domain changes

Add domain lookup by intent patterns

Write domain factory that builds adapters from database config

Test: Add a test domain via database, verify it appears in registry

1.3 Registration Implementation
Implement ephemeral registration path (no storage)

Implement persistent registration path (uses query_results)

Add automatic query retry after registration

Test: Discover cinema domain, approve, verify subsequent queries work

1.4 Admin Endpoints
Build GET /admin/discovery endpoint

Build POST /admin/discovery/:id/approve with override support

Build POST /admin/discovery/:id/reject

Add basic authentication to admin routes

Create simple admin UI (HTML or JSON view)

1.5 Cleanup
Remove old per-domain table creation code

Archive crime-specific code that's no longer needed

Update all tests to use new registry

Document how to add a domain without code

Phase 2: Output Generators — Multi-Modal Results
Goal: Support video, audio, 3D, and itinerary outputs
Duration: 6-10 weeks (can be parallelized)

2.1 Output Generator Infrastructure
Define OutputGenerator interface

Build output router that selects generator by query type

Create output_artifacts and artifact_snapshots tables

Add artifact storage to workspace system

2.2 Video Generator (Remotion)
Evaluate Remotion for server-side rendering

Create basic Remotion template for tutorials

Build asset sourcing from Unsplash/other APIs

Implement video render queue (async)

Add video export and sharing

Test: Generate tutorial video for a query

2.3 Audio Generator (Magenta/Tone.js)
Evaluate Magenta.js or Magenta API

Build composition parameter extraction from intent

Implement MIDI generation pipeline

Add Tone.js preview component

Test: Generate song from text description

2.4 3D Scene Generator (Three.js)
Evaluate React Three Fiber setup

Build sentiment-to-scene parameter mapping

Create reusable 3D scene components

Implement GLTF export

Test: Generate scene from sentiment data

2.5 Itinerary Generator
Build greedy interval scheduling algorithm

Integrate travel time API (Google Maps)

Add travel time caching

Build itinerary UI with timeline view

Test: Generate Fringe show itinerary

Goal: Extract core capabilities into composable tools
Duration: 2-3 weeks

3.1 Core Data Tools
Create fetch_data tool (existing, wrap as tool)

Create filter_by_constraints tool

Create rank_by_preference tool with scoring

Create group_by_location tool for clustering

3.2 Spatial Tools
Create calculate_travel tool with Google Maps integration

Create calculate_reachable_area tool (isochrones)

Add train-specific reachable area for UK rail

Precompute travel times for common domains (Edinburgh Fringe venues)

3.3 Optimization Tools
Create optimize_schedule tool (greedy + DP options)

Create sequence_by_geography tool (traveling salesman approximation)

3.4 Procedural Tools

3.5 Scraping Tool

4.5.5 Workspace Enhancement

Phase 5: Autonomous Discovery — Learning System
Goal: Agent discovers new domains and tools automatically
Duration: 3-4 weeks

5.1 Autonomous Discovery Agent
Create capability gap detection (agent knows when it can't answer)

Build discovery trigger from failed queries

Implement auto-approval for low-risk domains (confidence > 0.9)

Add risk assessment for discovered domains

Create discovery audit trail

5.2 Tool Discovery
Build tool proposal from discovery results

Implement tool auto-registration for high-confidence proposals

Create tool registry table

Add tool versioning for updates

Test: Agent discovers and registers new tool for carbon footprint

5.3 Learning Loop
Track successful and failed queries

Identify patterns in successful tool compositions

Update system prompts based on successful patterns

Suggest new tools for repeated patterns

Create learning dashboard for admin

5.4 Feedback Collection
Add user feedback buttons (helpful/not helpful)

Store feedback with query and response

Use feedback to adjust confidence thresholds

Create feedback review queue for admin

Test: Feedback improves future responses

5.5 Monitoring
Add telemetry for domain discovery events

Monitor tool usage patterns

Track relationship discovery quality

Set up alerts for failed discoveries

Create observability dashboard

Migrate crime and weather data — Ensure existing functionality continues

Build basic Domain and DataSource models — Start with schema, not full logic

Write admin approval endpoints — Simple GET/POST, authentication later

Once these are complete, the system is at a point where you can add new domains without code. From there, you can prioritize which output generators or tool library items make sense for your first use cases (cinema itineraries, hunting licenses, or game herd maps).
